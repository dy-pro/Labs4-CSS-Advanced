## Learning Objectives

Through this lab, I learned:

- How to build **responsive web layouts** using CSS without relying on JavaScript.
- Applying **mobile-first design principles**:
  - Starting with clean, simple styles for small screens.
  - Using relative units (`%`, `rem`, `vw`, `vh`) for scalable design.
- Using **media queries** to adapt layouts:
  - Defining breakpoints for tablet (`min-width: 768px`), desktop (`min-width: 1024px`) and wide screen ((`min-width: 1440px`)).
  - Adjusting font sizes, spacing, and layout structure across screen sizes.
- Designing structured layouts with **CSS Grid**:
  - Setting up `display: grid`, `grid-template-areas`, `grid-template-columns`, and `gap`.
  - Reorganizing grid columns responsively with media queries.
- Creating adaptive layouts with **Flexbox**:
  - Leveraging `flex: 1 1 calc(...)` for proportional widths.
  - Understanding the role of `flex-grow`, `flex-shrink`, and `flex-basis`.
  - Using `justify-content: center` to keep cards aligned neatly when the last row has fewer items.
- Comparing **Flexbox vs Grid**:
  - Flexbox for flexible, fluid layouts where the number of items per row can change.
  - Grid for precise, structured layouts with fixed column counts.
- Handling edge cases in responsive design:
  - Ensuring consistent card widths across rows.
  - Centering the last card(s) when the number of items doesn’t fill the row.
- Enhancing semantic HTML structure:
  - Using `<section>`, `<dl>`, `<dt>`, `<dd>`, and `<figure>` for meaningful markup.
  - Keeping headings (`<h2>`, `<h3>`) full width while cards align in rows..

---

This lab helped me practice **Advanced CSS Concept** (responsivenes, modern web layouts, media queries, mobile-first styling, CSS Grid and adaptive design principles) to design layouts that adapt seamlessly across different screen sizes and devices.
